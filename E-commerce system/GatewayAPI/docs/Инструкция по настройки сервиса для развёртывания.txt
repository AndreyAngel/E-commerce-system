Инструкция по настроки сервиса для развёртывания

1. Перейдите в файл appsettings.json.

2. Сервис использует SQLite3 в качестве СУБД. В стоке подключения к базе данных (DataBaseConnection) укажите название базы данных (параметр Database).

3. Укажите настройки сервера Kestrel (секция Kestrel):

		Ограничения (секция Limits):
			максимальное количество одновременных подключений (MaxConcurrentConnections);

			максимальный размер тела запроса в байтах (MaxRequestBodySizeInBytes);

			минимальная скорость передачи данных тела запроса (секция MinRequestBodyDataRate):
							кол-во байт в секунду (BytesPerSecond);
							льготное время, предоставленное пользователю для увеличения скорости соединения в секудах (GracePeriodInSeconds);

			минимальная скорость передачи данных ответа (секция MinResponseDataRate):
							кол-во байт в секунду (BytesPerSecond);
							льготное время, предоставленное пользователю для увеличения скорости соединения в секудах (GracePeriodInSeconds);

		Адрес приложения укажите в секции Endpoints/Https в строке Url.

4. Настройка брокера сообщений RabbitMQ (секция RabbitMQ):
		
		Url адрес сервера (Host);
		Имя пользователя (Username, по умолчанию guest)
		Пароль (Password, по умолчанию guest)

5. Откройте файл Ocelot.json. В нём находятся 2 секции: Routes для маршрутизации по сервисам и SwaggerEndPoints для корректной работты Swagger.
   Обе секции содержат по 3 словаря, один словарь для настройки одного сервиса.

6. В секции Routes заполните поля Host и Port - адрес сервиса и порт соответсвенно.

7. В секции SwaggerEndPoints заполните поля Url - расположение Swagger документации сервиса
